FEATURE: Color-Coded Help Output
---------------------------------

The ECO Checker now displays help text with beautiful color coding for improved
readability and user experience!

COLORS USED
-----------

Section Headers:
  - CYAN: "usage:", "positional arguments:", "options:", "Examples:"
  
Content:
  - GREEN: Tool description
  - YELLOW: All option flags (--workarea, -w, --help, etc.)
  - MAGENTA: "Author:" section
  - WHITE: Regular text, descriptions

USAGE
-----

Display colorized help:
  /home/avice/scripts/avice_wa_review/eco_checker.csh --help
  /home/avice/scripts/avice_wa_review/eco_checker.csh -h

Display help WITHOUT colors (plain text):
  /home/avice/scripts/avice_wa_review/eco_checker.csh --help --no-color
  /home/avice/scripts/avice_wa_review/eco_checker.csh -h --no-color

IMPLEMENTATION DETAILS
----------------------

1. Custom ColorHelpFormatter Class:
   - Extends argparse.RawDescriptionHelpFormatter
   - Overrides _format_usage(), _format_action(), and format_help()
   - Dynamically checks if colors are enabled

2. Color Management:
   - Colors automatically disabled when --no-color flag is present
   - Early detection: --no-color checked before parser creation
   - Uses existing Color class from eco_checker_lib.py

3. Smart Coloring:
   - Uses regex with word boundaries to avoid partial matches
   - Sorts option strings by length (longest first) to prevent conflicts
   - Colors applied consistently across all sections

TECHNICAL IMPLEMENTATION
------------------------

File: eco_checker.py

Key Components:
- ColorHelpFormatter class (lines 94-148)
- Early --no-color detection in main() (lines 204-207)
- Uses Color class: RED, GREEN, YELLOW, CYAN, MAGENTA, RESET

Color Codes (ANSI):
- CYAN:    \033[96m
- GREEN:   \033[92m
- YELLOW:  \033[93m
- MAGENTA: \033[95m
- RESET:   \033[0m
